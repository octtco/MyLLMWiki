# 03-review-gate

这页保留的是首轮 review gate 记录，主要用来说明“主干版为什么已经能先落页”。终版结论请看 [[07-终版-review-gate]]。

## 概念粒度检查

- `系统`、`对象边界`、`系统运动模型`、`状态`、`质量指标` 都通过了“能否脱离整条主题链单独解释”这道关。
- `反馈回路`、`可观测性`、`可控制性` 继续保留为正式概念页，但正文改成只守最小定义和接口，不再偷主题页的活。
- `系统类型`、`状态表示`、`相空间` 这轮故意不升正式页，避免把方法页里的关键区分拆成过碎概念。

## 局部组织检查

- 主题页先回答“面对复杂对象时为什么不能一上来就乱加动作”，再顺着这股压力把[[系统]]、[[状态]]、[[反馈回路]]、[[可观测性]]、[[可控制性]]一段段带出来，不再做成平行卡片堆。
- 方法页统一改成“卡点 -> 最容易错的地方 -> 这一步真正补什么 -> 什么时候调用 -> 场景变形 -> 边界”这条线。
- 概念页统一压回“最小定义 -> 常接到哪里 -> 不负责什么 -> 谁会调用它 -> 边界”，避免概念页重演整条主题主线。

## 叙事异味检查

- 旧问题是“正文讲事情是一套，链接又是一套”；这轮把关键链接都移到第一次真正需要理解的位置。
- 方法页不再靠文末的“相关概念”来补正文明明没讲透的地方。
- 主题页和概念页即便把文末链接区拿掉，正文主线也还能单独读下去。

## 仍然保留的风险

- 这 6 条方法和这组概念，目前仍主要由[[工程控制论（第三版）]]单一来源支撑；迁到项目、学习、组织等场景时，仍应保持低强度表述。
- 第三章到第十九章这轮没有细拆，所以后续还能继续补更细的方法页或概念页。
- `稳定性`、`时滞`、`噪声` 等后续概念是否正式升页，要等下一轮章节细拆后再判断。

## 当前结论

这轮可以正式落页，而且重点不只是“页型没错”，而是正文已经基本回到“先把事情讲通，链接只是防止读者在关键处卡住”的状态。
