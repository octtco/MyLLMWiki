# The Prompting Book 章节工作卡

## Introduction

- 位置：全书入口，交代作者、社区来源和新版动机。
- 要解决的问题：为什么 prompt 不是一次性技巧，而是一种可以被共享、积累和修订的实践。
- 直接出现的信号：Awesome ChatGPT Prompts、prompts.chat、社区贡献、模型变化。
- 最稳 takeaway：这本书的背景不是纯教程，而是 prompt 资产化社区的一次整理。
- 可复用但暂不说满：社区规模可以说明 prompt 作为资产有传播性，但不能证明每条 prompt 质量稳定。
- 前后连接：它从社区来源进入基础原则，为后面把 prompt 拆成结构件做铺垫。

## Foundations

- 位置：全书的最小工作定义层。
- 要解决的问题：用户如果不知道模型怎样响应文本，就容易把 prompt 写成含糊愿望。
- 直接出现的信号：LLM、上下文、role、context、task、constraints、format、examples、tone、clarity。
- 最稳 takeaway：有效 prompt 通常由角色、背景、任务、约束、格式和示例等构件组成，能被系统化组合。
- 可复用但暂不说满：这些构件可迁到多数文本模型任务，但具体权重仍受模型和任务影响。
- 前后连接：先有构件，后面的技巧才不是孤立咒语。

## Techniques

- 位置：把基础构件变成常见操作。
- 要解决的问题：同一个任务在没有角色、格式、示例和迭代时，输出容易飘散、难解析、难复现。
- 直接出现的信号：role-based prompting、structured output、chain of thought、few-shot learning、iterative refinement、JSON/YAML。
- 最稳 takeaway：这组技巧的共同作用，是把模型输出从自由文本往可控、可比、可改的形态推。
- 可复用但暂不说满：CoT 和 few-shot 等技巧在不同模型上的显式写法会变化，不能固定成万能公式。
- 前后连接：这组技巧把基础 prompt 变成可调试对象，再引出更复杂的系统和工作流设计。

## Advanced Strategies

- 位置：从单条 prompt 进入应用与系统层。
- 要解决的问题：真实工作不只需要一次回答，还需要长期角色、步骤串联、异常处理、上下文供给和可复用能力包。
- 直接出现的信号：system prompts、personas、prompt chaining、edge cases、multimodal prompting、context engineering、agents、skills。
- 最稳 takeaway：当任务复杂度上升，prompt 会成为 agent 和 skill 的基础材料，而不是完整系统本身。
- 可复用但暂不说满：书中 `prompts -> skills -> agents` 的比喻适合补充 [[Skill]]，但还不能单独定义所有 agent 架构。
- 前后连接：这一部分直接接到现有 [[AI coding framework]] 和 [[AI 工具化知识工作流]]，说明 prompt 资产进入系统后需要上下文、执行和验收纪律。

## Best Practices

- 位置：质量、责任和优化层。
- 要解决的问题：能产出不等于可靠、低风险、低成本或可维护。
- 直接出现的信号：pitfalls、ethics、responsible use、prompt optimization、quality/cost/speed trade-offs。
- 最稳 takeaway：提示词设计不是只追求一次漂亮输出，还要处理失败输入、偏见、误导、成本和一致性。
- 可复用但暂不说满：这些原则需要结合场景治理，不能直接当成单一安全规范。
- 前后连接：它给用例章节加边界，避免把 prompt 写成无约束万能工具。

## Use Cases

- 位置：场景落地层。
- 要解决的问题：同样的 prompt 构件在写作、编程、教育、商业、创意和研究里会变形。
- 直接出现的信号：writing、programming、education、business、creative arts、research。
- 最稳 takeaway：用例章节适合当场景样本池，帮助读者看见相同构件如何换场景。
- 可复用但暂不说满：单书用例不足以支撑每个场景的正式方法页。
- 前后连接：这些用例可为后续真实任务验证提供起点，而不是本轮直接拆案例页。

## Conclusion

- 位置：全书收束。
