# outputs

这里放运行过程中的工作层和任务结果，不要求一开始就写得像正式 wiki。它更像中间层和结果层，而不是知识库本体。

常见类型：

- `reports/`：问答、分析、综述、规程样板
- `slides/`：Marp 幻灯片
- `figures/`：图表说明、绘图配套文档
- `_templates/`：输出模板

长材料专项里，这里还承接：

- 范围定义
- 整体骨架梳理
- 章节工作卡
- 候选概念 / 方法 / 案例 / 主题池
- 当前边界清单

维护原则：

- 把 `outputs/` 当成合法工作层，不当成临时垃圾桶
- 长书、长报告和高密度长材料，先在这里过一层再回写 `wiki/`
- 不稳的迁移判断先留在这里
- 优先把高价值输出沉淀下来
- 需要长期复用时，再回写到 `wiki/`
- 建议给每个输出标注状态，区分草稿、可回写、已归档

推荐状态：

- `draft`：刚产出，还主要服务这次任务
- `reusable`：内容较稳，值得后续回写或拆分
- `promoted`：稳定部分已经沉淀进 `wiki/`
- `archived`：留档，不再继续维护
