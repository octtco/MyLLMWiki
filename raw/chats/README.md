# raw/chats

这里放聊天、会议、私聊摘录、语音转写这类对话型原始材料。

它仍然属于 `raw/`，默认按原始来源处理：

- 先保原文
- 先保时间和参与人
- 先保语境
- 不在这里写知识性总结

当前建议先把所有对话型材料统一放进 `raw/chats/`。以后如果会议、访谈、客服记录已经多到值得单独拆目录，再细分也不晚。

推荐结构仍然优先用“来源包”，不要把聊天文件散着放：

```text
raw/chats/
  2026-04-24-某次对话/
    transcript.md
    attachments/
    metadata.md
```

`metadata.md` 或 `metadata.json` 里建议至少保留：

- 参与人
- 时间范围
- 来源渠道
- 敏感级别或引用限制
- 这是不是完整记录，还是节选

维护原则：

- 不要一进 `raw/chats/` 就开始抽概念或写方法
- 对话型材料首先是来源，不是正式知识页
- 后续要提炼时，优先在 `wiki/sources/` 里生成聊天来源页，再决定更新哪些对象页、目标页或正式知识页
- 如果内容主要是行动推进，而不是知识沉淀，后续应优先回到 `todo/` 而不是硬塞进 `wiki/`
