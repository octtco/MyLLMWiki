# 首页

这是 MyLLMWiki 的入口。这页只回答一个问题：**回到这座库时，该从哪里下手。**

## 三种场景，三条路

### 1. 手上有新东西（剪藏、链接、聊天记录、PDF、仓库……）

扔进 `Clippings/`，然后让维护流程处理。不需要自己判断它该去哪一层——分流是维护流程的活。

### 2. 想找东西 / 问问题

从下面的索引入口往下钻，或者直接向维护助手提问（它会带出处回答）：

- [[主题索引]]：按问题找（最常用）
- [[来源索引]]：按资料找
- [[方法索引]] / [[概念索引]] / [[案例索引]] / [[领域索引]] / [[关系索引]]
- [[对象索引]] / [[目标索引]]：按项目和方向找

### 3. 想知道"我该干什么"

只看 [[待处理]]。它只放真打算做的事（当前 9 条）。
[[验证债]] 是按需取用的存档，**不是任务清单**，不用清零。

## 这座库的结构（一句话版）

新材料进 `Clippings/` → 原件保进 `raw/`（只读、保原貌）→ 编译成 `wiki/` 正式页（来源 / 对象 / 主题 / 方法 / 概念 / 案例 / 领域 / 关系）→ 行动归 `todo/`，工作底稿和报告归 `outputs/`，控制面在 `_index/`。不确定去哪的先放 `seeds/`。

两条 2026-06-10 起生效的运行规则：

- **入库分级**：新仓库 / 工具线索默认先登记进 [[样本池]]（一行一条），真正读过、用过或被论证需要时才升全套来源包 + 来源页
- **验证缺口**只写在来源页 `当前边界`，不自动生成待办

完整规则见 [[维护协议]]，运行状态见 [[总览]]，每轮干了什么见 [[log|运行日志]]。
