# wiki

这里放编译后的知识页面。

目录分工：

- `domains/`：按长期问题域组织的导航页
- `topics/`：围绕具体问题展开的入口页
- `methods/`：可复用的方法页
- `sources/`：来源摘要页
- `concepts/`：概念页
- `cases/`：案例页
- `relations/`：重要关系页
- `_templates/`：模板

维护原则：

- 从 `raw/` 出发，逐步沉淀结构化知识
- 当前默认主路径是：`领域页 -> 主题页 -> 方法页 -> 概念页 / 案例页 -> 来源页`
- 页面之间尽量建立显式链接
- `wiki/` 和 `_index/` 里的内部页面链接优先使用 Obsidian `[[wikilink]]`，避免空格或中文路径导致跳转不稳
- 严格 KG 语境下，关系页是正式页面类型；但只有在证据足够、关系本身值得展开时才建，不为了凑图谱而硬建
- 不为了“写满”而写，优先保留信息密度
