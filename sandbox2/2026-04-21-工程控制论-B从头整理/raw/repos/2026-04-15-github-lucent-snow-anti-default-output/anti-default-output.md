# anti-default-output

## 为什么有这个 skill

AI默认总是趋向于概率最大的输出，而我们知道最平均的输出往往就是保守的，没有特色的，这种情况导致AI输出的风格在无强提示词约束的情况下总是有很严重的“AI味”，在前端会体现为紫色渐变，小说可能会有事无巨细的细节描写，头脑风暴中可能是给两个极端选项，然后推荐第三个中庸的选择。我TM每次看到都要被气疯了。

因此我参考了，frontend-design skill，做了这个skill。

## 这个 skill 做了什么

它做的事很简单：

在输出之前，先让模型停一下，识别自己是不是正在走默认路径。

如果答案是“是”，那就不要直接把第一版交出去，而是主动换一个更有角度、更有判断、更不那么像模板的方向。

具体来说，就是提示词工程，告诉ai明显错误的范例，让AI去避免他。

这个仓库本身的结构也很简单：

- `SKILL.md` 是主文件，讲这个 skill 的核心思路和使用方式。
- `references/` 里放的是分领域的参考文件，用来总结某个场景下常见的 AI 默认错误。
- 现在已经有 `writing.md`、`fiction.md` 和 `frontend-design.md` 三个方向。

所以如果你之后想补内容，最直接的入手方式其实就是往 `references/` 里加新场景，或者继续完善已有文件。

## 如何使用

该skill可以在任何需要创意输出的情景下使用。

将其放入对应软件的skills文件夹下，ai会在需要时自动调用。

该skill主要是传达一种思路，你也可以将其整理进你的AGENTS.md中或者为你的情景专门设计一个skill。这些都是对这个仓库的正确使用方法。

## 结尾

如果你在特定的场景下，能够总结出特定的AI输出风格，欢迎提PR，将其整理进references文件夹下。

该skill现在采取的是告诉AI不该做什么，而没有告诉AI应该做什么，为了避免限制AI的输出多样性。这个判断还需要通过测试验证。

在这个前沿AI被训练地逐渐不说人话的当下，这个skill已经是我对AI公司最大的反抗了。f*** you ，oai！！！
