# outputs

这里放运行过程中产出的结果，不要求一开始就写得像正式 wiki。它更像任务结果层，而不是知识库本体。

常见类型：

- `reports/`：问答、分析、综述
- `slides/`：Marp 幻灯片
- `figures/`：图表说明、绘图配套文档
- `_templates/`：输出模板

维护原则：

- 优先把高价值输出沉淀下来
- 需要长期复用时，再回写到 `wiki/`
- 建议给每个输出标注状态，区分草稿、可回写、已归档

推荐状态：

- `draft`：刚产出，还主要服务这次任务
- `reusable`：内容较稳，值得后续回写或拆分
- `promoted`：稳定部分已经沉淀进 `wiki/`
- `archived`：留档，不再继续维护
