# Kimi - WebBridge 功能页

## 一句话总结

这份功能页最值得记住的，不是“agent 会逛网页”，而是它把浏览器控制明确拆成一层可复用基础设施。

## 当前范围

本页只整理 WebBridge 官方功能页，不补 Kimi 其他模型页、社区教程或实测视频。

## 摘要

- WebBridge 由浏览器扩展、本地服务和给 agent 调用的接口组成。
- 页面明确列出的兼容对象包括 Kimi Code、Claude Code、Cursor、Codex。
- 它承接的动作是打开网页、点击、输入、抽取内容等浏览器层操作。
- 这对当前知识库的价值，是把“浏览器控制桥”和“完整 agent 框架”区分开。

## 证据或原文线索

- 官方页标题：`WebBridge`
- 页面明确展示了 extension、本地桥接和 AI 编码助手的连接关系。
- 页面示例能力聚焦浏览器操作，而不是完整任务框架。

## 当前可支持的判断

- 当前来源支持：浏览器控制已经可以被封成给多个 agent 共用的一层桥接能力。
- 当前可保守迁出：在讨论 agent 能力时，应把“执行体”和“浏览器桥”分层，而不是混成一件事。

## 当前边界

- 这页不能单独支持“WebBridge 自己就是完整 agent 框架”。
- 这页也不支持“能复刻任意网页”这类扩张说法；它更直接支持的是浏览器操作和内容抽取。
