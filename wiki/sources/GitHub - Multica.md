# GitHub - Multica

## 一句话总结

Multica 是一个管理 coding agents 的开源平台样本，重点不是单个 agent 会写代码，而是把 agent 当成可分派、可跟踪、会沉淀技能的队友。

## 当前范围

本页整理可访问的 `multica-ai/multica`，用户原始线索里的 `nari-labs/multica` 当前精确访问为 404；不安装平台，不验证 daemon 或 self-hosting。

## 摘要

- README 把 Multica 定位为 managed agents platform。
- 核心口径是 assign tasks、track progress、compound skills。
- README 把 agent 放在 board、conversation 和状态更新里，而不是只放在本地终端。
- 它和 [[GitHub - andrej-karpathy-skills]] 属于同一组织下的互补样本：一个是平台，一个是规则文件。

## 证据或原文线索

- GitHub API 抓取时显示 stars 约 32.6k，仓库未归档。
- README 标题区写着 `The open-source managed agents platform` 和 `Turn coding agents into real teammates`。

## 当前可支持的判断

- 当前来源支持：coding agents 正在从单次运行工具，向可管理的队友 / worker 基础设施演化。
- 当前可保守迁出：任务分配、进度跟踪、技能沉淀是 managed agents 平台的三个核心抓手。

## 当前边界

- 不验证平台真实稳定性。
- `nari-labs/multica` 暂不能作为来源，只把 `multica-ai/multica` 作为当前可访问仓库。
