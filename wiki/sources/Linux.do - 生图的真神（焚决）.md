# Linux.do - 生图的真神（焚决）

## 一句话总结

这篇帖子本质上是一条高约束 Prompt 样本：它把“博物馆图鉴式中文拆解信息图”压成一段完整指令，当前最适合先当作风格模板来源保存。

## 摘要

- 正文几乎全部是一段成型 Prompt，核心做法是把主题自动映射成主体对象、材质、纹样、色彩、版式和中文标注系统。
- 这条 Prompt 最值得保留的部分，不是某个单独关键词，而是它对画面区块、信息密度、中文可读性和“图鉴气质”的强约束。
- 目前它更像一个可直接复用的提示词标本，而不是已经足够支撑上位方法页的稳定来源簇。

## 证据或原文线索

- 正文把版式明确固定为顶部标题区、左侧结构拆解区、右上材质工艺区、右中纹样色彩区、底部流程图区，说明它的核心不是灵感，而是结构约束。
- Prompt 明确要求“所有文字必须为简体中文，清晰、规整、可读”，这让它和普通海报型提示词明显不同。
- 结尾的避免项把海报感、影楼感、电商感、动漫感、乱标注、糊字、假材质都列成负面约束，说明它是在主动对抗模型默认风格。

## 我的判断

- 这份来源当前最适合保留为一条风格化 Prompt 样本。
- 它单独还不足以稳定支撑“焚决”或“风格化提示词设计”这种正式概念 / 方法页。
- 如果后面继续积累更多同簇来源，它可以作为一个很好的案例起点。
