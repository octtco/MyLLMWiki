# GitHub - Personal AI Infrastructure

## 一句话总结

Personal AI Infrastructure 是 Daniel Miessler 的个人 AI 基建设想和实现入口，强调以文件系统、context scaffolding、memory、skills 和自我改进循环放大人的能力。

## 当前范围

本页只整理 README 和 GitHub API 元数据；未部署项目，未验证 Life OS、Digital Assistant、filesystem context、memory、skills 或 self-improvement loop。

## 摘要

- README 的核心口径是 “AI should magnify everyone—not just the top 1%”。
- PAI 被描述为 Life OS，而不是普通 agent harness；原则包括 humans first、Ideal State、single Digital Assistant。
- 特性包括 text over opaque storage、context scaffolding、filesystem as context/no RAG、memory that compounds、self-improvement loop、skills as deterministic units 和 thinking skills。
- GitHub API 抓取时显示 TypeScript 仓库，MIT license，stars 约 15481，仓库未归档。

## 当前可支持的判断

- 当前来源支持：PAI 是个人 AI 基础设施 / life OS 方向样本。
- 它补充 [[AI 工具化知识工作流]]：个人知识工作可以被组织成文件系统、技能、记忆和循环改进的长期基建，而不是单个 app。
- 它也补充 [[Agent 框架设计]] 和 [[Agent 记忆与知识图谱]]：agent 的记忆和技能要接回人的目标、文件系统和可持续反馈。

## 当前边界

- README 带有明显理念宣言，当前不能把所有主张当作已验证架构。
- “filesystem as context, no RAG”和“single Digital Assistant”需要具体实现、规模和隐私边界来验证。
- 当前只保存为个人 AI 基建样本，不直接迁成 MyLLMWiki 改造方案。
