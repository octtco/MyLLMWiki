# GitHub - RSSHub

## 一句话总结

RSSHub 是一个把各种网站和服务转换成 RSS feed 的开源网络，README 把它称为 “Everything is RSSible”。

## 当前范围

本页只整理 README 和 GitHub 元数据，不部署实例，不逐条核验 route，也不评价各站点抓取合法性。

## 摘要

- 用户给出的 `rsshub` 按 GitHub 搜索第一结果识别为 `DIYgod/RSSHub`。
- README 把 RSSHub 描述为由 5,000+ global instances 组成的 RSS network。
- 项目目标是把多种来源内容聚合成 RSS，依靠开源社区持续补 routes、features 和 bug fixes。
- README 提到它和 Folo、RSSHub Radar、RSSBud、RSSAid 等相关项目配合使用。
- 项目提供官方文档、部署入口和贡献入口。

## 证据或原文线索

- GitHub API 抓取时显示 AGPL-3.0 license，stars 约 44.4k，仓库未归档。
- README heading 包括 `Introduction`、`Related Projects`、`Contribute`、`Deployment`。
- 仓库 homepage 指向 `https://docs.rsshub.app`。

## 当前可支持的判断

- 当前来源支持：RSSHub 是内容订阅和信息源聚合方向的重要基础设施样本。
- 当前可保守迁出：它适合进入“把分散网页变成可订阅 feed”的知识线，但具体站点 route 的稳定性和合规性仍需按场景复核。

## 当前边界

- 不扩写具体反爬、登录态或绕限制抓取方法。
- 不把 route 数量直接写成所有来源都稳定可用。
- AGPL-3.0 许可会影响服务部署和二次分发，后续使用需单独看许可边界。
