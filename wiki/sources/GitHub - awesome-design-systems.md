# GitHub - awesome-design-systems

## 一句话总结

这份 awesome list 最稳的价值，是把设计系统从“组件库”扩成一组可比较的公开样本：组件、语气文案、设计师套件和源码开放程度都被放在同一张表里。

## 当前范围

本页只整理 README 快照，不逐一打开列表里的每个设计系统，也不判断各项目当前是否仍可用。

## 摘要

- README 把 design system 定义成一组关于原则和最佳实践的文档，用来帮助团队构建数字产品。
- 列表同时承认 `design systems`、`ui libraries` 和 `pattern libraries` 在实际使用中经常被混用。
- 表格字段把样本拆成四类能力：Components、Voice & Tone、Designers Kit、Source code。
- 列表覆盖公司、政府、开源框架、内容风格指南等多种来源，适合作为横向找样本的入口。
- Apple Developer Design Guidelines 也被列入其中，且被标记为包含 Designers Kit。

## 证据或原文线索

- README 开头直接给出 design system 的定义，并说明它可以包括 UI libraries、pattern libraries、Voice and Tone。
- Tags 段落解释四个标记字段各自代表什么。
- Notes 段落提醒开源标记不等于一定可自由使用，仍需检查许可证。

## 当前可支持的判断

- 当前来源支持：设计系统资料不应只按“有没有组件”筛选，还要看是否有语言规范、设计资源和源码入口。
- 当前可以低强度迁出：如果要研究一个产品的设计系统成熟度，可以先用这四个维度做粗筛。

## 当前边界

- 这是社区维护的导航表，不保证每个链接仍然有效。
- 它能提供样本池，但不能替代对单个设计系统的深入阅读。
- 表格里的标记是 list 维护者口径，不等于官方自证。
