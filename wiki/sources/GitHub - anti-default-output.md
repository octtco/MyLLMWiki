# GitHub - anti-default-output

## 一句话总结

这组原始文件共同支持一种很明确的主张：当输出价值不只取决于正确性，还取决于角度、语气和判断时，模型需要先识别自己的默认路径，再决定要不要主动偏离。

## 摘要

- 主文件说明了这个 skill 的出发点：模型总会滑向高概率、保守、模板化的输出。
- `files/SKILL.md` 把这件事进一步讲成一个明确动作：生成前先停一下，判断自己是不是在走默认路径。
- `files/writing.md` 和 `files/frontend-design.md` 又把这种默认错误拆成更具体的识别信号。

## 证据或原文线索

- 主文件明确把“默认总是趋向于概率最大的输出”作为问题起点。
- `files/SKILL.md` 直接提出 `生成前的自检`，并要求模型先问“如果我不加约束，会不会写成所有 AI 都会给的那个版本”。
- `files/writing.md` 罗列了 AI 写作里的默认错误信号，`files/frontend-design.md` 给了前端设计里的默认审美清单。

## 我的判断

- 当前这组材料足够支持 [[反默认输出]]、[[默认路径]]、[[输出前自检]] 三个概念页。
- 它最强的地方不是给出统一审美，而是给出一种先识别默认错误、再主动偏离的工作方式。
- 但“这样做一定能稳定产出更高质量结果”目前仍主要是仓库主张，还缺更多外部样本支撑。
