# Apple - Human Interface Guidelines

## 一句话总结

Apple HIG 最稳的价值，是把 Apple 平台设计收成一套从平台归属感、基础元素、模式、组件、输入方式到技术集成的官方规范入口。

## 当前范围

本页只整理 HIG 首页和官方 DocC JSON 快照，不展开每个子页面，也不把当前所有 Apple 平台组件规范逐项搬入库。

## 摘要

- HIG 首页把目标写成帮助设计者为任何 Apple 平台创造优秀体验。
- 首页把顶层结构拆成 Getting started、Foundations、Patterns、Components、Inputs、Technologies。
- 首页给出三条原则：Hierarchy、Harmony、Consistency。
- Design fundamentals 列出 App icons、Color、Materials、Layout、Icons、Accessibility。
- New and updated 当前列出 Multitasking、The menu bar、Toolbars、Search fields、Game Center、Generative AI。
- 页面还链接到 `Adopting Liquid Glass`，说明 HIG 会随平台设计语言演进持续更新。

## 证据或原文线索

- 官方 meta description 写明 HIG 包含 guidance 和 best practices，用于设计 Apple 平台体验。
- DocC JSON 的 `topicSections` 直接列出六个顶层主题。
- DocC JSON 的 `primaryContentSections` 保存了 Hierarchy、Harmony、Consistency 三条首页原则。

## 当前可支持的判断

- 当前来源支持：面向 Apple 平台的产品设计不只是视觉风格问题，而是要先对齐平台结构、组件约定、输入方式和技术能力。
- 当前可以低强度迁出：当做 iOS / macOS / visionOS 等体验设计时，HIG 应作为官方约束源，而不是普通灵感站点。

## 当前边界

- HIG 是活文档，今天的快照不能代表未来页面内容不变。
- 首页只能支撑结构入口和原则口径，具体组件决策仍要回到对应子页面。
- `Adopting Liquid Glass` 等新材料需要另行整理后，才能写成更具体的设计判断。
