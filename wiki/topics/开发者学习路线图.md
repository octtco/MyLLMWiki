# 开发者学习路线图

## 这页真正想回答什么

这页想回答的是：当学习目标很大、方向很多时，路线图到底帮人解决什么问题。

## 为什么这个问题容易散

开发者学习很容易变成“看到什么学什么”。前端、后端、DevOps、AI Engineer、MLOps、System Design、Prompt Engineering、AI Agents 都有自己的关键词和工具链。如果没有一层路径组织，人会很快在资料收藏里获得安全感，却没有真正知道下一步该补什么。

## 当前最稳的主线

[[GitHub - developer-roadmap]] 当前支持的主线是：路线图的价值不是替代教材，而是把一个方向拆成可浏览、可选择、可补盲区的能力地图。README 里列出的 roadmaps、best practices 和 questions，分别对应三种不同的学习动作：先选路，再看实践约束，最后用问题测试掌握程度。

这也解释了它和 [[系统设计学习入口]]、[[机器学习工程入口]] 的关系。system design 或 machine learning 进入学习时，都很容易被名词淹没；路线图提供的是导航层，真正的工程判断仍要回到具体来源、项目和复盘里验证。

## 当前边界

- 当前只有 developer-roadmap 一个来源支撑，先不把每条路线拆成独立主题页。
- 路线图只适合作为入口和导航，不等于最佳学习顺序已经被验证。
- README 的 license 字段为 `NOASSERTION`，复用具体内容前仍需查许可。
