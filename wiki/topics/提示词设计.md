# 提示词设计

## 这页到底在回答什么

这页想回答的是：提示词什么时候只是一次性问法，什么时候开始变成可以复用、迭代、接入工作流的设计对象。

## 为什么这个问题会让人反复做散

提示词很容易被讲成两种薄东西：一种是“复制某条神奇 prompt”，另一种是“掌握一堆技巧名词”。前者忽略任务、上下文和失败边界，后者又容易把角色提示、结构化输出、CoT、few-shot、JSON、agent、skill 这些东西堆成术语列表。

## 当前最稳的主线

当前最稳的主线是：提示词设计先把任务说清，再把输出形态收住，随后通过示例、迭代和边界测试把结果变得可调试；当任务变复杂时，它就不再只是单条文本，而会进入上下文工程、链式流程、[[Skill]] 和 agent 这类系统层。

[[The Prompting Book]] 给了这条主线的完整骨架。它先讲模型和有效 prompt 的组成，再讲角色、结构化输出、思维链、few-shot、迭代、JSON/YAML，最后进入 context engineering、agents 和 skills。这个顺序提醒我们：所谓 prompt engineering 更稳的理解，不是背模板，而是逐步控制模型从“理解任务”到“稳定交付”的条件。

## 这条主线怎样一段一段展开

第一步是构件化。一个 prompt 至少要能说清角色、背景、任务、约束、输出格式和必要示例。缺哪一块，模型就会用默认路径补哪一块，所以结果看起来像“模型不听话”，实际常常是输入没有把任务边界交代完。

第二步是格式化。结构化输出、JSON 和 YAML 的价值，不只是让答案好看，而是把模型输出变成后续流程可以消费的形态。到了这里，提示词已经开始从聊天文本靠近工作流接口。

第三步是迭代化。few-shot、失败样本、边界输入和反复改写让 prompt 变成可测试对象。它不是一次写完，而是在观察输出后继续收紧任务、格式和约束。

第四步是系统化。当任务需要长期上下文、多个步骤、工具调用或可复用能力时，单条 prompt 就不够了。这里会接到 prompt chaining、context engineering、[[Skill]] 和 agent：prompt 仍是基础材料，但真正可靠的系统还需要上下文供给、触发时机、执行纪律和验收。

这也解释了为什么 [[GitHub - awesome-chatgpt-prompts]] 和 [[GitHub - awesome-ai-dev-prompts]] 仍然重要但不能被夸大。它们说明 prompt 可以被收藏、分享和资产化；但只有当这些文本被放回具体任务、验证流程和能力边界里，才会从素材库继续往方法或系统靠近。

[[GitHub - Prompt Engineering Interactive Tutorial]] 给这条线补了一个官方交互式练习样本：它不是单纯列技巧，而是把 basic structure、clear/direct、role、data/instruction separation、formatting、examples、hallucination control 和 complex prompts 做成课程与练习。这让本页可以更明确地区分“prompt 素材库”和“prompt 教学 / 调试工作流”。

视觉生成来源提供了另一个边界样本。[[Linux.do - 特殊风格焚决与 Prompt 思路]] 和 [[Linux.do - 生图的真神（焚决）]] 说明，提示词也可以主动对抗模型默认风格；[[Linux.do - GPT Image 2.0 生图 50 种风格速查表]] 又补了另一类约束：不只是写风格词，还要规定宫格、标题、注解、截断和视觉整齐优先级。但这类写法强依赖模型、审美目标和参考样本，当前仍更适合当风格化 prompt 样本，而不是通用设计法则。

## 当前边界

- 这页不是提示词技巧大全。
- 不把某个模板、技巧名或风格 prompt 写成跨模型稳定规律。
- 不把 prompt library 等同于 skill、agent 或完整 workflow。
- CoT、few-shot、context engineering 等候选概念先留在工作层，等更多来源和真实任务验证。
- Anthropic 交互式教程当前只按 README 保存，Notebook 练习和当前模型效果未复测。

## 后续可追的问题

- 选一个真实知识工作任务，对比“单条 prompt / prompt library / skill / agent workflow”的实际差异。
- 后续如果继续积累上下文工程和 prompt 评测来源，再判断是否单独拆概念页。
- 如果风格化生图来源继续增多，再评估是否从本页分出“风格化提示词设计”主题或方法页。
