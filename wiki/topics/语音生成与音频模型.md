# 语音生成与音频模型

## 这页真正想回答什么

这页先回答一个窄问题：开源语音模型进入 TTS、voice cloning、ASR 和本地推理工程时，知识库里应该怎样保存能力边界。

## 当前最稳的主线

[[GitHub - VoxCPM]] 当前给出的样本是 tokenizer-free TTS：README 里把 VoxCPM2 描述成 2B 参数、多语言、voice design、controllable cloning、ultimate cloning 和 48kHz 输出的语音生成模型。

这和 [[音频分离数据集与监督纯度]] 的问题不同。Hive 这条线更关心音频分离数据集和监督信号纯度；VoxCPM 这条线更关心语音合成、声音设计和声音克隆。两者都属于音频模型，但暂时分开维护，避免把生成、分离、数据集和版权风险混成一页。

[[GitHub - whisper.cpp]] 把这页从 TTS / voice cloning 扩到 ASR 和本地推理工程。README 支持的稳妥判断是：whisper.cpp 关注的是 Whisper 自动语音识别模型的 C/C++ 高性能推理、量化、跨平台和多硬件后端，而不是语音生成本身。

## 当前边界

- 当前只有 VoxCPM 和 whisper.cpp 两个来源，先不扩成语音模型综述。
- voice cloning 只应在授权、同意、身份安全和合成内容标注边界内讨论。
- README 的质量、语种、延迟和商业可用口径未复测。
- ASR 转写涉及录音隐私和授权边界，whisper.cpp 的性能也需要按模型、硬件和音频格式复测。
