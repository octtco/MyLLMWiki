# 算法与数据结构学习入口

## 这页真正想回答什么

这页想回答的是：算法和数据结构资料进入知识库时，先应该作为学习入口保存，还是立刻拆成一堆概念页。

## 为什么这个问题容易散

算法材料天然容易拆成名词表：数组、链表、树、图、排序、搜索、动态规划、贪心。拆得太快会让知识库看起来很完整，但实际阅读时很难知道学习顺序、练习方式和复杂度判断该怎么连起来。

## 当前最稳的主线

[[GitHub - JavaScript Algorithms]] 当前更适合作为代码样本池和学习入口，而不是立即拆概念。它用 JavaScript 实现常见数据结构和算法，并在 README 里附上 Big O、复杂度表和相关参考。

这条线可以补 [[开发者学习路线图]]：roadmap 负责告诉你算法和数据结构在开发者成长路径中的位置，具体仓库则提供可阅读、可运行、可对照的实现样本。

## 当前边界

- 当前只有一个来源支撑，先不拆算法概念页。
- 不验证每个实现的正确性。
- 不把 JavaScript 实现当成所有语言学习的唯一入口。
