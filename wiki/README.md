# wiki

这里放编译后的正式知识页面。

目录分工：

- `sources/`：来源摘要页
- `entities/`：对象页，承接人、项目、产品、组织、长期对象
- `topics/`：围绕问题的主题页
- `methods/`：可复用的方法页
- `concepts/`：概念页
- `cases/`：案例页
- `domains/`：领域页
- `relations/`：重要关系页
- `_templates/`：模板

补充说明：

- 不是每一轮都要把所有页型建齐
- 一般先有来源页，再看是否需要对象页，最后再往下筛 `主题 -> 方法 -> 概念 / 案例`
- 长材料先在 `outputs/` 建工作层，再回到这里落正式页
- 目标和待办不在这里维护，而在 `todo/` 里维护

维护原则：

- 从 `raw/` 出发，逐步沉淀结构化知识
- 页面之间尽量建立显式链接
- 页型职责尽量分开，不让对象页、主题页、方法页、概念页互相偷活
- `wiki/` 和 `_index/` 里的内部页面链接优先使用 Obsidian 的 wikilink 语法，避免空格或中文路径导致跳转不稳
- 严格 KG 语境下，关系页是正式页面类型；但只有在证据足够、关系本身值得展开时才建，不为了凑图谱而硬建
- 不为了“写满”而写，优先保留信息密度
