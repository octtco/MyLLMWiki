# MyLLMWiki

## 这是什么

`MyLLMWiki` 是当前这座基于 Obsidian 的个人知识库项目本体。它不是单篇笔记集合，而是一个有明确分层、工作流和控制面的持续整理系统：新材料先进入 `Clippings/`，再按状态分流到 `raw/`、`seeds/`、`todo/`、`outputs/` 和 `wiki/`。

## 为什么它现在重要

因为你后面的目标已经不是“存几篇资料”，而是让网页、书、聊天记录、灵感、目标、待办，最后都能慢慢长成只属于你的个人知识网络。这个仓库就是那条主线的实际落点。

## 当前状态

- 结构层面已经补齐到 `Clippings / raw / seeds / wiki / todo / outputs / assets / _index` 这套新版口径。
- 正式来源层已经跑出一批真实样板，长材料、网页来源、聊天批次归档和项目级聊天切片都开始进入运行态。
- 对象层和目标层已经不再只是目录、模板和协议层；这一轮开始有真实对象页、目标页和项目级聊天来源页一起联动。
- 当前仍有不少来源停在保守表达和待复查状态，说明这座库已经开始区分“先接住”和“再抽象”这两件事。

## 关键关系

- 它当前最直接的推进目标，是 [[把 MyLLMWiki 跑成可持续增长的个人 LLM Wiki]]。
- 它和 [[知乎专栏 - Karpathy 的 LLM Wiki 新范式]] 的关系，更像“目标参照”而不是“现成照抄”。
- 它和 [[Codex rollout 历史聊天归档（2026-02 至 2026-04）]] 的关系，是“母语境档案”；而 [[Codex rollout - MyLLMWiki 项目切片（2026-04-13 至 2026-04-23）]] 则是更靠近项目本体的历史切片。
- 它现在的维护方式，和 [[先把当前状态和伏笔单独记账，再续写长文本]] 这条方法很接近：先把当前状态外置，再决定下一轮该读什么、改什么。

## 最近变化

- 统一收件箱 `Clippings/` 已经正式上线为默认入口。
- `raw/chats/` 已经补出第一份真实批次归档样板，并从里面切出第一份 `MyLLMWiki` 项目级子归档。
- `$myllmwiki-steward` 的 skill 和 reference 已经追到新版结构口径。
- 这轮又新进了一组 Linux.do 的小说写作系列来源，并补出了相应来源页、主题页和方法页。

## 当前未决问题

- rollout 聊天归档现在已经为 `MyLLMWiki`、`WeBox`、`WhatsBox`、`Beam` 跑出四份项目样板；但当前明确继续跟进的，先只留 `MyLLMWiki`、`WeBox`、`WhatsBox` 这三条。
- 对象层和目标层已经开始长样板，但还缺更多真实材料来验证稳定写法。
- 一些高波动论坛帖和单作者经验帖，后面仍要继续补交叉来源，防止抽象过满。
