# seeds

这里放候选素材。

它的职责是接住那些“值得留一下，但现在还不适合直接放进 `raw/`、`wiki/`、`outputs/` 或 `todo/`”的内容。

常见情况：

- 偶然看到的小工具、小方法、小技巧
- 一段暂时还不算正式来源的摘录或线索
- 带一点个人判断的记录，比如“为什么这个东西值得留”
- 还没想好最终该归到哪一层的混合材料

它不是这些东西：

- 不是原始来源归档层：能明确作为正式来源保存的，优先进 `raw/`
- 不是正式知识页：已经沉淀稳定的内容，应该进 `wiki/`
- 不是任务结果层：一次问答、分析、报告，应该进 `outputs/`
- 不是待办层：明确要做的事情，应该进 `todo/`

目录约定：

- `inbox/`：默认入口，新内容先放这里
- `_templates/`：种子卡模板

推荐做法：

1. 先把新东西记成一张短卡，别急着第一次就分对类
2. 卡片里保留“原始内容 + 为什么值得留 + 可能去向”
3. 后面再按成熟度分流到 `raw/`、`wiki/`、`outputs/` 或 `todo/`

推荐命名：

- `YYYY-MM-DD-关键词.md`
- 或者直接用能看懂的语义名，但尽量别过长
