# Vibecoding：是真的吗与帮助学习两个方向

## 原始内容

- 用户明确说：`两个 idea：是真的吗 / 帮助学习` 是准备 vibecoding 的两个方向，不需要抽象出来。
- 相关材料：
  - [Abnerla/AI_paper](https://github.com/Abnerla/AI_paper)：一站式 AI 学术论文写作辅助工具，覆盖定题、出稿、降 AIGC、查重率、润色、纠错等环节。
  - [YANZHANLIN/ielts-claude-skills](https://github.com/YANZHANLIN/ielts-claude-skills)：雅思备考 AI 教练 Claude Code skills，覆盖写作、阅读、口语训练。

## 为什么值得留

- 这是用户自己的项目方向，不是普通外部资料。
- `是真的吗` 和 `帮助学习` 都适合 vibecoding：方向足够具体，可以先做很小的原型，不必先写成大而全的平台。
- 两个方向都能吃到 skill / agent / 检索 / 结构化反馈这些已有线索，但最终应该服务项目落地，而不是变成抽象知识页。

## 现在的直觉

- `是真的吗` 更像事实核验产品方向：输入一个说法、链接、截图或短视频片段，输出可读的核验结果、来源链、可信度和未证实点。
- `帮助学习` 更像学习陪练方向：输入目标、当前水平和材料，让 agent 生成练习、纠错、复盘和下一步任务。
- `AI_paper` 更靠近“写作辅助 + 学术流程”；`ielts-claude-skills` 更靠近“考试学习 + skill 化教练”。它们可以作为参考对象，但不直接定义这两个方向。

## 可能去向

- `todo/goals`：如果开始正式做 vibecoding 项目，需要建目标页，写清楚先做哪个方向、做到什么算完成。
- `todo/tasks`：拆第一批原型任务，比如“做一个是真的吗 MVP 输入输出流”或“做一个 IELTS/通用学习反馈 demo”。
- `wiki/entities`：如果这两个方向变成具体项目名，再建对象页。
- `outputs/reports`：可先写一份“两个方向的 MVP 对比与优先级判断”。
