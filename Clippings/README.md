# Clippings

这里现在作为**统一收件箱**使用。

默认做法是：

- 你先把新来的原始资料放进 `Clippings/`
- 再由维护流程把它分流到 `raw/`、`seeds/`、`todo/` 或其他更合适的位置

它不是长期存储层，也不是正式知识层。它更像“还没分流前的入口”。

## 适合先放进来的东西

- 网页剪藏
- 聊天记录导出
- 会议纪要原文
- 语音转写原文
- PDF、书、论文、仓库说明
- 截图、图片、附件
- 一时还没想好该去哪的混合材料

## 默认分流方向

- 明确是网页 / 文章来源 -> `raw/web/`
- 明确是聊天 / 会议 / 转写 -> `raw/chats/`
- 明确是书 / 论文 / 仓库 / 数据集 / 图片 -> `raw/` 下对应目录
- 还没熟、但值得保留 -> `seeds/`
- 明确属于目标或待办 -> `todo/`

也就是说，`Clippings/` 本身不负责“整理完”，它只负责**先接住**。

## 命名原则

在 `Clippings/` 里不用一开始就命名得很工整。

更重要的是：

- 先别丢内容
- 先别破坏原文
- 先让资料进库

真正的标准命名，放到分流进 `raw/`、`seeds/`、`todo/` 时再做。

## 清理原则

一份材料只要已经成功迁到正式落点，并且核对无误，就可以把 `Clippings/` 里的原文件删掉。

所以这里不是归档层，而是**可被清空的收件箱**。
