# todo

这里是独立于知识层的行动控制区。

- `goals/`：目标页，负责方向、阶段、卡点和推进策略
- `tasks/`：待办页，负责当前可执行动作
- `待办事项.base`：Obsidian Base 视图，当前主要按 `tasks/` 使用
- `_templates/`：目标和待办的可复用模板

建议做法：

1. 长期或中期方向先在 `todo/goals/` 建目标页
2. 目标页里挂相关对象、相关知识和下一步任务
3. 当前能执行的动作再拆到 `todo/tasks/`
4. 完成后的阶段分析先放 `outputs/`，稳定后再回写 `wiki/`

一个简单判断：

- 如果你主要在回答“为什么现在做、做到哪了”，先去 `goals/`
- 如果你主要在回答“下一步具体做什么”，先去 `tasks/`
- 如果你主要在回答“这个东西现在是什么、为什么重要”，那它大概率不在 `todo/`，而在 `wiki/entities/`
