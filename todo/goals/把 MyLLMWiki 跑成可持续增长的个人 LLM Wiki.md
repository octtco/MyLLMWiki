# 把 MyLLMWiki 跑成可持续增长的个人 LLM Wiki

## 目标一句话

让网页、书、聊天、灵感、目标和待办都能先进库、被正确分流，并逐步长成真正可复用的个人知识网络。

## 为什么现在做

因为结构和 skill 口径现在已经基本对齐了，下一步最需要的不是再讲规则，而是继续拿真实材料把对象页、目标页、聊天归档二次切分这些新版能力跑顺。

## 做到什么算完成

- 新材料进入 `Clippings/` 后，能稳定分流到 `raw / seeds / todo / outputs / wiki`。
- 对象层、目标层、聊天来源页都不再只是模板，而是有足够真实样板可参考。
- 重点长材料、聊天归档和多来源主题都能走完整工作流，而不是中途停在半成品。
- 控制面能持续同步，后面每轮维护都不会重新回到“先猜结构”的状态。

## 当前阶段

阶段性完成

## 当前卡点

- 作为“让 MyLLMWiki 跑成个人 LLM Wiki”的第一阶段目标，当前已经可以收口。
- 仍需持续观察的，不是这条目标是否成立，而是后续具体材料进入仓库时能否继续稳定执行分流、回写和自检。
- 某些高波动来源仍只有单作者或单站经验，后面继续按 `review` / `seed` 口径保守处理。

## 推进策略

- 先继续用真实材料补小样板，而不是先追求大而全。
- 优先处理和当前仓库本身关系最强的材料，比如聊天归档、结构升级输出和新进来源簇。
- 能保守表达的先保守表达，不为了凑页型硬拔高。
- 每轮都把状态、队列和日志回写清楚，避免知识库自己也陷入长文本失忆。
